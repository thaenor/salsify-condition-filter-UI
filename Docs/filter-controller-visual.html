4 actions (dispatched to filterReducer)

selectProperty(id) → needs-operator, clears operator + value + error
selectOperator(id) → needs-value (or → ready if any/none), clears value + error
setValue(raw) → ready if parse OK, stays needs-value + sets parseError if not
clear() → needs-property, everything resets

Reset cascade (why it feels complex)

Change property → operator gone, value gone, error gone. Back to needs-operator.
Change operator → value gone, error gone. Back to needs-value (or ready for any/none).
Change value → parse OK? ready. Parse fail? stays needs-value + error.
Clear → everything gone. Back to needs-property.
Each level resets everything below it. That's the whole rule.
